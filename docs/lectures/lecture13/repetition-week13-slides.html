Why Vector Fields Matter

Physics is full of vector quantities that vary in space:

Field Symbol Units
Electric field \(\vec{E}(\vec{r})\) N/C
Magnetic field \(\vec{B}(\vec{r})\) T
Velocity field \(\vec{v}(\vec{r})\) m/s
Force field \(\vec{F}(\vec{r})\) N

Challenge: How do we visualize something that has both magnitude and direction at every point?

The Fundamental Tool: Meshgrid

Problem: We want to evaluate a function at many 2D points

Solution: Create coordinate grids with np.meshgrid()

x = np.linspace(-2, 2, 5)
y = np.linspace(-1, 1, 3)

X, Y = np.meshgrid(x, y)

Result: Two 2D arrays containing all combinations of x and y coordinates

Understanding Meshgrid: Visual

x = np.array([0, 1, 2])
y = np.array([0, 1])

X, Y = np.meshgrid(x, y)

X array:

[[0, 1, 2],
 [0, 1, 2]]

Y array:

[[0, 0, 0],
 [1, 1, 1]]

Each position (i,j) in X and Y gives you a coordinate pair!

Meshgrid: The Concept

Think of it as a grid of points:

y
^
1 | (0,1)  (1,1)  (2,1)
  |
0 | (0,0)  (1,0)  (2,0)
  +-------------------> x
    0      1      2

Key insight: X[i,j] and Y[i,j] together specify one point on this grid

Why Meshgrid?

Without meshgrid (painful):

for i, xi in enumerate(x):
    for j, yj in enumerate(y):
        result[i,j] = function(xi, yj)

With meshgrid (vectorized):

X, Y = np.meshgrid(x, y)
result = function(X, Y)  # Operates on entire grid at once!

Benefits: Faster, cleaner, more readable

Exercise 1: Electric Field

Physics: Point charge at origin creates radial electric field

\[\vec{E}(\vec{r}) = \frac{q}{4\pi\epsilon_0} \frac{\vec{r}}{r^3}\]

In 2D Cartesian coordinates:

\[E_x = \frac{kq \cdot x}{r^3}, \quad E_y = \frac{kq \cdot y}{r^3}\]

where \(r = \sqrt{x^2 + y^2}\)

Challenge: Visualize this field!

The Plan: Electric Field

  1. Create a grid of points in the x-y plane
  2. At each point, calculate the distance \(r\) from origin
  3. Calculate the x and y components of \(\vec{E}\)
  4. Use plt.quiver() to draw arrows

Key function: plt.quiver(X, Y, Ex, Ey) - X, Y: positions of arrow tails - Ex, Ey: arrow directions (field components)

Quiver Plots: Concept

plt.quiver(X, Y, Ex, Ey)

What it does: - At position (X[i,j], Y[i,j]) - Draw an arrow - Pointing in direction (Ex[i,j], Ey[i,j]) - Length proportional to magnitude

Perfect for vector fields!

Step 1: Create the Grid

# Define coordinate ranges
x = np.linspace(-2, 2, 20)
y = np.linspace(-2, 2, 20)

# Create meshgrid
X, Y = np.meshgrid(x, y)

Result: 20√ó20 grid of points covering the region -2 ‚â§ x ‚â§ 2, -2 ‚â§ y ‚â§ 2

Step 2: Calculate Distance

# Distance from origin
R = np.sqrt(X**2 + Y**2)

Problem: What happens at the origin? Division by zero!

Solution: Add a tiny number

R = np.sqrt(X**2 + Y**2 + 1e-16)

Now R is never exactly zero, avoiding singularities

Step 3: Calculate Field Components

For a unit charge at origin:

# Electric field components (simplified, k*q = 1)
Ex = X / R**3
Ey = Y / R**3

Why this works: - \(\vec{r} = (X, Y)\) is the position vector - \(r^3 = R^3\) is the distance cubed - Field points radially outward: \(\vec{E} \propto \vec{r}/r^3\)

Step 4: Plot the Vector Field

plt.figure(figsize=(8, 8))
plt.quiver(X, Y, Ex, Ey)
plt.xlabel('x position (m)')
plt.ylabel('y position (m)')
plt.axis('equal')
plt.show()

Result: Arrows pointing radially outward from the origin!

Complete Electric Field Code

x = np.linspace(-2, 2, 20)
y = np.linspace(-2, 2, 20)

X, Y = np.meshgrid(x, y)

R = np.sqrt(X**2 + Y**2 + 1e-16)

Ex = X/R**3
Ey = Y/R**3

plt.figure(figsize=(8, 8))
plt.quiver(X, Y, Ex, Ey)
plt.xlabel('x position (m)')
plt.ylabel('y position (m)')
plt.axis('equal')
plt.show()

Enhancing the Visualization

Optional: Add field magnitude as background

E_mag = np.sqrt(Ex**2 + Ey**2)

plt.figure(figsize=(8, 8))
plt.contourf(X, Y, E_mag, levels=20, cmap='viridis')
plt.colorbar(label='Field Magnitude (N/C)')
plt.quiver(X, Y, Ex, Ey, color='white')
plt.xlabel('x position (m)')
plt.ylabel('y position (m)')
plt.axis('equal')

Color shows magnitude, arrows show direction!

Exercise 2: Standing Waves

Physics: Vibrating membrane (drum head) has standing wave modes

\[\psi(x,y) = \sin(mx)\cos(ny)\]

Parameters: - \((m, n)\) ‚Äî mode numbers (integers) - Determine the pattern of nodes and antinodes

Examples: - \((1, 1)\) ‚Äî Fundamental mode - \((2, 3)\) ‚Äî Higher harmonic with complex pattern

What Are Standing Waves?

Standing wave: Combination of waves that doesn‚Äôt propagate

Nodes: Points that never move (œà = 0)

Antinodes: Points with maximum amplitude

Musical connection: - Different \((m,n)\) ‚Üí different notes - Higher modes ‚Üí higher pitches - Node patterns ‚Üí timbre (sound quality)

Visualizing 2D Functions

Two main approaches:

  1. Contour plot: View from above, color indicates value
    • plt.contour() ‚Äî contour lines
    • plt.contourf() ‚Äî filled contours
  2. Surface plot: 3D perspective
    • ax.plot_surface() ‚Äî 3D surface

Best practice: Show both views!

Step 1: Create Fine Grid

# Need many points for smooth visualization
x = np.linspace(-np.pi, np.pi, 100)
y = np.linspace(-np.pi, np.pi, 100)

X, Y = np.meshgrid(x, y)

Note: More points (100) than vector field (20) for smooth contours

Step 2: Calculate Wave Pattern

# Mode numbers
m, n = 2, 3

# Wave function
psi = np.sin(m*X) * np.cos(n*Y)

Try different modes: - (1, 1) ‚Äî Simple pattern - (2, 3) ‚Äî Complex pattern - (3, 3) ‚Äî Symmetric pattern

Step 3: Create Contour Plot

plt.figure(figsize=(8, 8))
plt.contourf(X, Y, psi, levels=20, cmap='RdBu')
plt.colorbar(label='Amplitude')
plt.xlabel('x position')
plt.ylabel('y position')
plt.axis('equal')
plt.show()

Color scheme: - Red: Positive displacement - Blue: Negative displacement - White: Nodes (zero displacement)

Step 4: Add 3D Surface Plot

fig = plt.figure(figsize=(10, 8))
ax = fig.add_subplot(projection='3d')

surf = ax.plot_surface(X, Y, psi, cmap='RdBu')
ax.set_xlabel('x position')
ax.set_ylabel('y position')
ax.set_zlabel('Amplitude')
plt.colorbar(surf)

3D view shows: - Height = wave amplitude - Peaks and valleys clearly visible

Complete Standing Wave Code

x = np.linspace(-np.pi, np.pi, 100)
y = np.linspace(-np.pi, np.pi, 100)
X, Y = np.meshgrid(x, y)

m, n = 2, 3
psi = np.sin(m*X) * np.cos(n*Y)

plt.figure(figsize=(16, 8))

# 2D contour

plt.ylabel('y position')
plt.axis('equal')

# 3D surface
ax = plt.subplot(122, projection='3d')
ax.plot_surface(X, Y, psi, cmap='RdBu')
ax.set_xlabel('x position')
ax.set_ylabel('y position')
ax.set_zlabel('Amplitude')

plt.tight_layout()

Exercise 3: Quantum Wave Packet

Physics: Localized quantum particle described by Gaussian wave function

\[\psi(x,y) = A\exp\left(-\frac{(x-x_0)^2 + (y-y_0)^2}{2\sigma^2}\right)\]

Parameters: - \((x_0, y_0)\) ‚Äî center position (where particle is likely) - \(\sigma\) ‚Äî width (uncertainty in position) - \(A\) ‚Äî amplitude (normalization)

Quantum Interpretation

Wave function œà: Describes quantum state

Probability density: \(P(x,y) = |\psi(x,y)|^2\)

Physical meaning: - \(P(x,y)\) gives probability of finding particle at (x,y) - Larger \(\sigma\) ‚Üí more uncertain position - Peak at \((x_0, y_0)\) ‚Üí most likely location

Heisenberg uncertainty principle: Can‚Äôt know position exactly!

The Gaussian Function

Why Gaussian?

  1. Smooth ‚Äî No sharp edges
  2. Localized ‚Äî Falls off exponentially
  3. Well-behaved ‚Äî Easy to normalize
  4. Natural ‚Äî Appears everywhere in physics
  5. Minimum uncertainty ‚Äî Saturates Heisenberg limit

Step 1: Create the Grid

x = np.linspace(-5, 5, 100)
y = np.linspace(-5, 5, 100)

X, Y = np.meshgrid(x, y)

Range from -5 to 5 to capture most of the wave packet

Step 2: Set Parameters

x0, y0 = 1.0, 0.0  # Center position
sigma = 1.0        # Width (standard deviation)
A = 1.0            # Amplitude

Try different values: - Move center: change (x0, y0) - Widen packet: increase œÉ - Narrow packet: decrease œÉ

Step 3: Calculate Wave Function

# Wave function
psi = A * np.exp(-((X-x0)**2 + (Y-y0)**2)/(2*sigma**2))

# Probability density
prob_density = np.abs(psi)**2

For real œà: \(|\psi|^2 = \psi^2\)

For complex œà: \(|\psi|^2 = \psi^* \psi\) (use np.abs())

Step 4: Visualize Three Ways

  1. Wave function amplitude ‚Äî œà itself
  2. Probability density ‚Äî |œà|¬≤
  3. 3D surface ‚Äî See the Gaussian peak
plt.figure(figsize=(15, 5))

plt.subplot(131)
plt.contourf(X, Y, psi)
plt.title('Wave Function')

plt.subplot(132)
plt.contourf(X, Y, prob_density)
plt.title('Probability Density')

ax = plt.subplot(133, projection='3d')
ax.plot_surface(X, Y, prob_density)

Complete Wave Packet Code

x = np.linspace(-5, 5, 100)
y = np.linspace(-5, 5, 100)
X, Y = np.meshgrid(x, y)

x0, y0 = 1.0, 0.0
sigma = 1.0
A = 1.0

psi = A * np.exp(-((X-x0)**2 + (Y-y0)**2)/(2*sigma**2))
prob_density = np.abs(psi)**2

plt.figure(figsize=(15, 5))

plt.subplot(131)
plt.contourf(X, Y, psi, levels=20, cmap='RdBu')
plt.xlabel('x'); plt.ylabel('y')
plt.title('Wave Function')
plt.axis('equal')

plt.subplot(132)
plt.contourf(X, Y, prob_density, levels=20, cmap='viridis')
plt.xlabel('x'); plt.ylabel('y')
plt.title('Probability Density')
plt.axis('equal')

ax = plt.subplot(133, projection='3d')
ax.plot_surface(X, Y, prob_density, cmap='viridis')
ax.set_xlabel('x'); ax.set_ylabel('y')
ax.set_zlabel('Probability')

Colormap Choices Matter

Common colormaps:

  • 'viridis' ‚Äî Perceptually uniform, good default
  • 'RdBu' ‚Äî Red-blue diverging, good for ¬±values
  • 'hot' ‚Äî Heat map
  • 'coolwarm' ‚Äî Another diverging option

Rules of thumb: - Diverging (RdBu): Data with positive and negative values - Sequential (viridis): Data with one polarity - Always add colorbar!

3D Plotting: Projection

To create 3D plots:

fig = plt.figure()
ax = fig.add_subplot(projection='3d')

# or

ax = plt.subplot(111, projection='3d')

Then use: - ax.plot_surface() ‚Äî Surface plots - ax.plot_wireframe() ‚Äî Wireframe - ax.contour3D() ‚Äî 3D contours

Common Pitfalls

Problem Cause Solution
Arrows too small/large Scale issue Use scale parameter in quiver
Division by zero Singularities Add 1e-16 to denominator
Blocky contours Too few levels Increase levels= parameter
Wrong grid size X.shape ‚â† Y.shape Check meshgrid output
Distorted plots Unequal axes Use plt.axis('equal')

Debugging Meshgrid Issues

Check array shapes:

print(X.shape, Y.shape)  # Should be the same!
print(psi.shape)         # Should match X and Y

Visualize the grid:

plt.plot(X, Y, 'k.')  # Plot all grid points

Check a single point:

print(f"Point: ({X[0,0]}, {Y[0,0]})")

Performance Tips

For large grids:

  1. Use appropriate resolution (balance quality vs. speed)
  2. Avoid explicit loops ‚Äî use vectorized operations
  3. For animations, reuse figure objects
  4. Use sparse sampling for quiver plots (every Nth point)
# Sample every 2nd point for faster quiver
plt.quiver(X[::2, ::2], Y[::2, ::2], 
           Ex[::2, ::2], Ey[::2, ::2])
