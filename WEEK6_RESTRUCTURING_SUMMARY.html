    Week 6 Restructuring Summary

    Overview

    This document summarizes the reorganization of Week 6 materials to improve pedagogical flow and student learning outcomes. The restructuring was based on Option 1: adding an application week between numerical methods (Week 5) and code organization (Week 7).

    Lecture Order: Planetary Motion (easier) ‚Üí Spring Pendulum (more complex/chaotic)

    Problem Identified

    The original course structure had a pedagogical gap between Week 5 and Week 6:

    Original Structure (Problematic)

    • Week 5: Numerical Methods (differentiation, ODE solving, integration)
    • Week 6: Code Organization (modules, OOP, file I/O)

    Issues

    1. Lost Momentum: Students learned powerful ODE-solving tools but didn‚Äôt immediately apply them to exciting physics problems
    2. Motivational Gap: The shift from physics tools to software engineering concepts broke the flow
    3. Missed Opportunity: Commented-out content (planetary motion, spring pendulum) would have been perfect applications

    Solution Implemented

    New Structure (Improved)

    • Week 5: Numerical Methods Toolkit
      • Numerical differentiation
      • Solving ODEs with odeint
      • Numerical integration
    • Week 6: Classical Mechanics Applications ‚≠ê NEW
      • Planetary Motion (Newton‚Äôs laws & Kepler)
      • Spring Pendulum & Chaos
    • Week 7: Code Organization
      • Modules
      • Classes & OOP
      • Advanced Brownian Motion with OOP
      • File I/O

    Students now use ODE-solving tools immediately after learning them: - Week 5: Learn odeint - Week 6: Apply odeint to exciting physics problems - Week 7: Learn to organize complex code (now motivated by need)

    Planetary motion and spring pendulum share: - Same coordinate system (polar) - Similar equation structure - Same solution methodology - Different force laws demonstrate generality - Progression from stable (planets) to chaotic (spring) shows range of behaviors
