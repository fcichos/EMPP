Why Differential Equations?

Physics is written in the language of differential equations:

Equation Physics
\(F = ma = m\frac{d^2x}{dt^2}\) Newton‚Äôs mechanics
\(\frac{dN}{dt} = -\lambda N\) Radioactive decay
\(\frac{d^2\theta}{dt^2} = -\frac{g}{L}\sin\theta\) Pendulum
\(i\hbar\frac{\partial\Psi}{\partial t} = \hat{H}\Psi\) Quantum mechanics

Problem: Most of these cannot be solved analytically!

Analytical vs. Numerical Solutions

Analytical solution: Exact formula (when possible)

\[N(t) = N_0 e^{-\lambda t}\]

Numerical solution: Step-by-step approximation

\[N(t + \Delta t) \approx N(t) + \Delta t \cdot \frac{dN}{dt}\]

Key insight: We trade exactness for the ability to solve any ODE!

The Foundation: Taylor Series

Any smooth function can be expanded around a point \(t_0\):

\[f(t) = f(t_0) + (t-t_0)f'(t_0) + \frac{(t-t_0)^2}{2!}f''(t_0) + \frac{(t-t_0)^3}{3!}f'''(t_0) + \ldots\]

What does each term represent?

  • \(f(t_0)\) ‚Äî value at the starting point
  • \((t-t_0)f'(t_0)\) ‚Äî linear change (slope)
  • \(\frac{(t-t_0)^2}{2!}f''(t_0)\) ‚Äî curvature correction
  • Higher terms ‚Äî finer corrections

Taylor Series: Visual Intuition

Euler method: Keep only the first two terms (linear approximation)

The Euler Method: Core Idea

If we truncate Taylor series after the linear term:

\[f(t_0 + \Delta t) \approx f(t_0) + \Delta t \cdot f'(t_0)\]

In words:

‚ÄúThe new value equals the old value plus the change‚Äù

‚ÄúChange = slope √ó time step‚Äù

This is the Euler method ‚Äî the simplest ODE solver!

Euler Method: Single Step Visualized

Euler Method: Step by Step

Given: \(\frac{dy}{dt} = f(y, t)\) with initial condition \(y(0) = y_0\)

Algorithm:

  1. Start with \(y_0\) at \(t = 0\)
  2. Calculate the slope: \(\text{slope} = f(y_0, t_0)\)
  3. Take a step: \(y_1 = y_0 + \Delta t \cdot \text{slope}\)
  4. Repeat from the new point
for j in range(1, num_steps):
    slope = derivative(y[j-1], t[j-1])
    y[j] = y[j-1] + dt * slope

Example: Radioactive Decay

The decay equation:

\[\frac{dN}{dt} = -k N\]

What we know:

  • \(N\) = number of radioactive nuclei
  • \(k\) = decay constant
  • The derivative depends on the current value of \(N\)

Applying Euler:

\[N(t + \Delta t) = N(t) + \Delta t \cdot (-k N(t))\]

Translating Math to Code

The math: \[\frac{dN}{dt} = -kN\] \[N_{new} = N_{old} + \Delta t \cdot (-kN_{old})\]

The code:

def decay(N, k):
    """The derivative: dN/dt = -kN"""
    return -k * N

def euler_step(N, dt, k):
    """One step of Euler method"""
    return N + dt * decay(N, k)

Notice: The code directly mirrors the math!

The Complete Euler Solver

# Define the derivative
def decay(N, k):
    return -k * N

# Setup
t = np.arange(0, 5, 0.1)  # time points
dt = t[1] - t[0]           # time step
N = np.zeros(len(t))       # storage for results
N[0] = 100                 # initial condition

# Solve!
for j in range(1, len(t)):
    N[j] = N[j-1] + dt * decay(N[j-1], k=1)

That‚Äôs it! You‚Äôve just built an ODE solver.

How Accurate is Euler‚Äôs Method?

Error: Proportional to \(\Delta t\) (first-order method) ‚Äî smaller steps = better accuracy!

Improving Accuracy: The Key Insight

Euler‚Äôs problem: Uses slope only at the start of the interval

Better idea: Use the average slope across the interval!

\[f(t_0 + \Delta t) \approx f(t_0) + \Delta t \cdot \frac{f'(t_0) + f'(t_0 + \Delta t)}{2}\]

But wait‚Ä¶ We don‚Äôt know \(f'(t_0 + \Delta t)\) yet!

Solution: First predict using Euler, then correct using the average.

Improved Euler Method (Heun‚Äôs Method)

Step 1 ‚Äî Predict: Use regular Euler to estimate the endpoint

\[y_{pred} = y_n + \Delta t \cdot f'(y_n)\]

Step 2 ‚Äî Correct: Average the slopes at start and predicted endpoint

\[y_{n+1} = y_n + \frac{\Delta t}{2}\left(f'(y_n) + f'(y_{pred})\right)\]

Result: Error is now proportional to \((\Delta t)^2\) ‚Äî much better!

Improved Euler: Visual Intuition

Improved Euler in Code

def improved_euler(y, dt, derivs, *params):
    # Step 1: Predict using basic Euler
    y_pred = y + derivs(y, *params) * dt
    
    # Step 2: Average the slopes
    slope_start = derivs(y, *params)
    slope_end = derivs(y_pred, *params)
    average_slope = 0.5 * (slope_start + slope_end)
    
    # Step 3: Take the step with average slope
    return y + dt * average_slope

Key difference: Two derivative evaluations per step instead of one.

Second-Order ODEs: The Challenge

Many physics problems involve second derivatives:

\[\frac{d^2y}{dt^2} = -\omega^2 y \quad \text{(harmonic oscillator)}\]

Problem: Our Euler methods work on first-order ODEs!

Solution: Convert to a system of first-order ODEs.

The Trick: Introduce Velocity

Original equation: \[\frac{d^2y}{dt^2} = -\omega^2 y\]

Define: \(v = \frac{dy}{dt}\) (velocity)

Now we have two first-order equations:

\[\frac{dy}{dt} = v\] \[\frac{dv}{dt} = -\omega^2 y\]

State vector: \(\vec{s} = \begin{pmatrix} y \\ v \end{pmatrix}\)

Visualizing the State Vector

Harmonic Oscillator: The Derivative Function

def harmonic_oscillator(state, omega):
    """
    state = [y, v] = [position, velocity]
    Returns [dy/dt, dv/dt] = [v, -omega¬≤¬∑y]
    """
    y, v = state
    dydt = v
    dvdt = -omega**2 * y
    return np.array([dydt, dvdt])

Notice: The function takes a vector and returns a vector!

Solving the Harmonic Oscillator

# Initial conditions: y=1, v=0 (released from displacement)
state = np.zeros((len(t), 2))
state[0] = [1, 0]

# Time evolution
for j in range(1, len(t)):
    state[j] = improved_euler(state[j-1], dt, 
                               harmonic_oscillator, omega=1)

# Extract position and velocity
y = state[:, 0]
v = state[:, 1]

The solver doesn‚Äôt change! Only the derivative function changes.

Coupled Equations: Predator-Prey

Some systems have variables that influence each other:

\[\frac{dx}{dt} = \alpha x - \beta xy \quad \text{(prey)}\] \[\frac{dy}{dt} = \delta xy - \gamma y \quad \text{(predators)}\]

The Lotka-Volterra equations:

  • Prey grow exponentially (\(\alpha x\)) but are eaten (\(-\beta xy\))
  • Predators die without food (\(-\gamma y\)) but reproduce when fed (\(\delta xy\))

Understanding the Interactions

  1. More prey ‚Üí more food ‚Üí predators increase
  2. More predators ‚Üí more hunting ‚Üí prey decrease
  3. Less prey ‚Üí predators starve ‚Üí predators decrease
  4. Fewer predators ‚Üí less hunting ‚Üí prey recover
  5. ‚Ä¶ cycle repeats!

This creates oscillations in both populations!

Lotka-Volterra: The Code

def lotka_volterra(state, alpha, beta, gamma, delta):
    """
    state = [x, y] = [prey, predators]
    """
    x, y = state
    
    dxdt = alpha*x - beta*x*y    # prey dynamics
    dydt = delta*x*y - gamma*y   # predator dynamics
    
    return np.array([dxdt, dydt])

Same structure as before! Vector in, vector out.

Lotka-Volterra: Results

The closed loop in phase space is called a limit cycle ‚Äî the system oscillates forever!

What About scipy.odeint?

You‚Äôve used scipy.integrate.odeint in previous lectures:

from scipy.integrate import odeint

solution = odeint(derivative_function, initial_condition, t_array)

What does it do internally?

  • Uses Runge-Kutta methods (like improved Euler, but even better)
  • Automatically adjusts step size for accuracy
  • Handles stiff equations

Now you understand the principle! odeint is just a fancier version of what you built today.

Comparison of Methods

Method Error Evaluations/step Use case
Euler \(O(\Delta t)\) 1 Learning, simple problems
Improved Euler \(O(\Delta t^2)\) 2 Better accuracy
Runge-Kutta 4 \(O(\Delta t^4)\) 4 Standard choice
scipy.odeint Adaptive Varies Production code

Rule of thumb: Use scipy for real work, but understand Euler to debug!

Common Mistakes to Avoid

Mistake Symptom Fix
\(\Delta t\) too large Solution explodes or oscillates Reduce step size
Wrong initial conditions Wrong starting point Check units and values
Derivative sign error Solution grows instead of decays Check your physics!
Forgetting to update Solution stays constant y[j] = ... not y = ...

Debugging Strategy for ODEs

  1. Test with known solution ‚Äî Compare to analytical result
  2. Check energy conservation ‚Äî Does total energy drift?
  3. Vary \(\Delta t\) ‚Äî Solution should converge as \(\Delta t \to 0\)
  4. Plot intermediate steps ‚Äî See where things go wrong
# Energy check for harmonic oscillator
E = 0.5 * v**2 + 0.5 * omega**2 * y**2
plt.plot(t, E)  # Should be constant!

Summary: The Recipe for Solving ODEs

  1. Write the ODE ‚Äî What‚Äôs \(\frac{dy}{dt} = f(y, t)\)?
  2. Define the derivative function ‚Äî def f(y, params): return ...
  3. Set initial conditions ‚Äî What‚Äôs \(y(0)\)?
  4. Choose time array and step size ‚Äî t = np.arange(0, T, dt)
  5. Loop and update ‚Äî y[j] = y[j-1] + dt * f(y[j-1])
  6. Plot and verify ‚Äî Does it make physical sense?
