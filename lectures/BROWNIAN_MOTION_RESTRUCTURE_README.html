    Brownian Motion content has been split into two modules to improve learning outcomes:

    • Week 2-3: Simple, function-based approach (NEW)
    • Week 6-7: Advanced OOP approach (EXISTING, moved from Week 2-3)

    This change reduces cognitive load, allows students to focus on physics first, and demonstrates the value of OOP through direct comparison.

    Why This Change?

    The Problem

    Original structure taught classes in Week 2-3 via Brownian Motion, causing:

    • ‚ùå Cognitive overload (too many new concepts at once)
    • ‚ùå Students confused by self, __init__, OOP paradigm
    • ‚ùå Physics understanding suffered
    • ‚ùå Students asked ‚Äúwhy do we need this?‚Äù (no motivation for OOP yet)

    The Solution

    Two-stage approach:

    Stage 1 (Week 2-3): Physics First - Students understand Brownian motion conceptually - Build confidence with functions, loops, arrays - See immediate results ‚Üí motivation boost - Natural programming progression

    Stage 2 (Week 6-7): Professional Code - Revisit familiar physics with better tools - Students SEE why classes help (direct comparison) - Natural motivation: ‚Äúmy Week 2 code was messy, this is better!‚Äù - Learning OOP makes sense in context

    Teaching Guide

    Week 2-3: Simple Brownian Motion

    File: lectures/lecture02/4_brownian_motion_simple.qmd

    Learning Goals: - Understand Brownian motion physics - Generate random steps with np.random.normal() - Store trajectories in arrays - Compute and plot mean squared displacement - Build confidence with functions

    Teaching Tips: - Emphasize: ‚ÄúWe‚Äôre focusing on physics first!‚Äù - Celebrate when students get their first random walk working - Allow experimentation with parameters (D, dt, N) - Don‚Äôt mention ‚Äúwe‚Äôll do this better later‚Äù - let them enjoy the success

    Common Questions: - Q: ‚ÄúWhy lists instead of something fancier?‚Äù - A: ‚ÄúKeeping it simple so we focus on understanding the physics!‚Äù

    Week 6-7: OOP Brownian Motion

    File: lectures/lecture05/2_brownian_motion.qmd

    Learning Goals: - Understand when to use classes vs. functions - Design and implement classes for physics simulations - Appreciate code organization benefits - Compare approaches critically
